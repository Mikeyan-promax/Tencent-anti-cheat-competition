[cite_start]**参赛者**：闫子凌 (赛事 ID: #1492) [cite: 164, 165, 374, 375]

## 1. 项目简介
[cite_start]本项目为 2026 腾讯游戏安全技术竞赛（AI方向）的端到端大模型行为预测与战术续写方案 [cite: 167, 373][cite_start]。针对初赛的“离散行为分类”和决赛的“连续语义生成 (Seq2Seq)”双重任务挑战，本方案以 Qwen2.5 系列模型为核心 [cite: 167, 377][cite_start]。在单卡 RTX 4090 算力受限的环境下，融合了数据降噪、知识蒸馏、Agentic CoT（代理思维链）、模型微调（SFT）以及端侧规则兜底等前沿技术，实现了高拟真度、零违禁词、强鲁棒性的游戏战术行为自动化分析 [cite: 167, 169, 377, 381]。

## 2. 初赛技术路线：离散行为序列预测
[cite_start]初赛阶段的核心任务是基于玩家过去20秒的高频对局日志，预测未来5秒的战术意图与具体动作 [cite: 172]。

* [cite_start]**高频日志降噪与并行特征提取**：针对高频数据易导致显存溢出 (OOM) 和注意力稀释的问题，执行了语义事件无损保留与坐标轨迹的整秒下采样，成功将文本冗余降低 90% 以上 [cite: 176, 178, 179, 207][cite_start]。辅以 16 核多进程并行流式处理架构 (chunksize=500)，极大提升了特征提取的吞吐量，将清洗耗时缩减至分钟级 [cite: 168, 208]。
* [cite_start]**极限 LoRA 监督微调 (SFT)**：在单卡 24GB 显存限制下，采用 Qwen2.5-7B-Instruct 作为基座，应用 4-bit NF4 双重量化与梯度检查点 (Gradient Checkpointing) 技术防爆显存 [cite: 167, 219, 220][cite_start]。设定超参数 LoRA $r=16, \alpha=32$，学习率 $2\times10^{-4}$，并引入步数为 8 的梯度累积，实现等效 Batch Size 16 的高效平滑收敛 [cite: 230, 231, 234]。
* [cite_start]**启发式规则兜底 (Heuristic Fallback)**：针对对局中“被救起/投掷”等罕见动作极度匮乏导致的长尾效应，利用大模型预测的高置信度大类意图（交战/避战）作为先验条件，对占绝大多数的模糊动作（如“移动”）进行二次语境推断（如细化为“靠近敌人”或“后撤”），成功修正了单一分类任务的业务盲区 [cite: 170, 247, 252]。

## 3. 决赛技术路线：连续语义生成与文本续写
[cite_start]决赛任务升级为基于对局文本续写后续行为描述，面临无标准答案、极易生成游戏梗语以及长上下文算力瓶颈三大难点 [cite: 377, 565][cite_start]。本方案确立了“14B 教师模型知识蒸馏 + 7B 学生模型特训 + 推理端闭环”的非对称双层技术路线 [cite: 413, 561]。

### 3.1 第一阶段：数据工程与离线知识蒸馏
* [cite_start]**分层抽样与长尾压制**：通过安全下限阈值的分层抽样策略，强制保留稀有战术类，将 9.1 万条存在严重类别不平衡的对局数据精准压缩至 10,000 条高质量均衡语料 [cite: 429, 451]。
* [cite_start]**作弊数据拦截 (Truncation Shield)**：针对高频脚本宏或外挂玩家引发的超长数据洪流（5秒日志超2000字符），系统触发强制硬截断并注入标准化拦截标识，彻底消除了上下文溢出导致的推理崩溃 [cite: 447, 456]。
* [cite_start]**14B 模型极速蒸馏**：本地独立部署 Qwen2.5-14B-Instruct-AWQ 配合 vLLM 引擎，全离线极速产出包含 Agentic CoT 的优质对齐数据，极大压缩了数据生成耗时 [cite: 415, 458]。

### 3.2 第二阶段：生成式微调与最优权重剥离
* [cite_start]**头部熔断保护**：为避免 HuggingFace 默认的尾部截断物理抹除目标监督信号 (`<report>`)，本方案引入基于时序特征的“头部熔断防线”，剥离前置冗余日志，实现 100% Target 零截断，同时缩减了约 60% 的计算耗时 [cite: 465, 466, 563]。
* [cite_start]**Loss 掩码重构**：引入指示函数重构自回归交叉熵损失，精准屏蔽冗长系统指令的梯度回传，使模型优化完全聚焦于战术生成的语义拟合 [cite: 468, 566]。
* [cite_start]**纯净权重剥离**：在模型稳健收敛（Eval Loss 稳步降低至 0.302）后触发自动净化流程，从最优 checkpoint 节点物理剥离出纯粹的 LoRA 推理权重，抛弃臃肿的优化器状态以释放端侧算力 [cite: 503, 566]。

### 3.3 第三阶段：端侧安全推理与防线闭环
[cite_start]作为直接偏好优化 (DPO) 容易引发 OOM 的轻量化平替，本方案在推理端部署了极低显存开销的双重防线 [cite: 528, 530, 567]：
* [cite_start]**词表级约束解码 (Constrained Decoding)**：构建黑名单词表探针，将“干拉”、“大身位”、“赛门”等非规范电竞梗语映射为 Token IDs。在底层的 Logit 概率计算阶段执行严格掩码（概率置为 $-\infty$），从物理树根节点阻断违禁词的生成 [cite: 538, 548]。
* [cite_start]**启发式特征兜底校验**：定制化部署 Python 校验流，一旦大模型生成的输出与前20秒物理日志硬指标发生背离（如出现“无源投掷幻觉”），立即触发强制降级纠偏（如降级为“进行战术空间拉扯”），实现自动化处罚的零误判 [cite: 550, 567]。
