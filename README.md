# 2026 腾讯游戏安全技术竞赛 (AI方向) - 全链路行为预测与续写方案

## 1. 项目概述
本项目针对《三角洲行动》游戏中的“GameGPT-生成式AI大模型”任务，构建了一套端到端的行为预测系统。任务核心在于通过分析玩家过去 20 秒的高频物理日志，精准预测其未来 5 秒的战术意图与具体动作。

本方案经历了从初赛的**“离散行为分类”**到决赛的**“连续语义生成（Seq2Seq）”**的完整演进。技术架构以 **Qwen2.5** 系列大模型为核心，创新性地提出了“非对称双层蒸馏架构”，在单卡 RTX 4090 的算力限制下，实现了高拟真、零幻觉、零违禁词的战术复盘生成。

---

## 2. 初赛技术路线：时序特征降噪与行为分类
初赛核心在于从极其冗余的原始日志中捕获深层时序逻辑（如“受击后丢烟”判定为避战）。

### 2.1 高频日志降噪引擎
* **语义事件强制保留**：无损保留“伤害”、“技能生效”、“物资点”等低频强表征事件。
* **轨迹整秒下采样**：针对每秒 20 帧的玩家坐标信息，仅保留整数秒位置。该策略将文本冗余度降低了 **90% 以上**，有效解决了 Token 爆炸与注意力稀释问题。
* **16核并行流式处理**：利用多进程架构，将数十万级样本的处理时间从数小时压缩至分钟级。

### 2.2 极限 LoRA 微调策略
* **显存压榨**：在 24GB 显存限制下，采用 4-bit NF4 量化与梯度检查点（Gradient Checkpointing）技术。
* **超参数组合**：选用 LoRA 秩 $r=16$、$\alpha=32$，学习率定为 $2 \times 10^{-4}$。通过 8 步梯度累积实现等效 Batch Size 16，确保 Loss 稳定收敛至 0.2。

## 3. 决赛技术路线：知识蒸馏与 Agentic CoT 生成
决赛要求输出自然语言形式的战术复盘。面对“无标准答案”与“算力瓶颈”双重挑战，本方案重构了研发流水线。

### 3.1 非对称双层蒸馏架构 (Teacher-Student)
由于直接微调大参数模型易导致显存崩溃，本项目采用了以下架构：
1.  **教师模型 (Teacher)**：部署 **Qwen2.5-14B-Instruct-AWQ**，利用 vLLM 引擎全离线生成包含推理链路（Agentic CoT）的高质量语料。
2.  **学生模型 (Student)**：利用生成的 10,000 条精品数据微调 **Qwen2.5-7B-Instruct**。
3.  **结果**：7B 模型跨越式继承了 14B 的逻辑推演能力，且推理速度提升 2 倍以上。

### 3.2 针对性数据工程
* **分层抽样 (Stratified Sampling)**：针对交战行为占比超 85% 的长尾分布，强制保留 100% 的稀有战术类样本（如“救援”、“丢雷”），拉平后验概率分布。
* **头部熔断保护 (Head Truncation)**：针对超长日志，物理剥离早期冗余（保留 `input_text[-2500:]`），确保末端的 `<report>` 监督信号不被截断，训练耗时缩减 60%。
* **作弊数据拦截 (Truncation Shield)**：自动识别 5 秒内封包数超 2000 的疑似外挂/脚本宏玩家，执行强制截断并注入安全拦截标识。

### 3.3 训练优化：Loss 掩码重构
在微调阶段，引入指示函数重构自回归损失函数，仅对目标回复序列（Target）计算梯度，屏蔽系统指令与原始日志的 Loss。该方法显著提升了模型对战术语义拟合的专注度。

## 4. 推理端安全闭环防线
为平替高资源消耗的 DPO 对齐，本方案在推理端部署了双重防御机制：

### 4.1 词表级约束解码 (Constrained Decoding)
* **原理**：在底层的 `generate()` 阶段挂载 `bad_words_ids` 探针。
* **管控对象**：将“赛门”、“刘涛”、“干拉”、“白给”等电竞梗语与非官方黑话的 Logits 概率置为 $-\infty$。
* **效果**：从物理层切断模型“越狱”的可能性，确保报告的专业性与规范性。

### 4.2 启发式幻觉兜底校验 (Heuristic Fallback)
建立逻辑校验规则库，实时对比模型生成文本与前 20 秒物理事实：
* **无源投掷校验**：若前 20 秒无物资搜刮且未持有道具，生成内容中若出现“丢雷”，则降级纠偏为“进行战术空间拉扯”。
* **无视野开火校验**：若可见性持续为 0 且无受击，将“果断开火”修正为“保持架枪预瞄”。
